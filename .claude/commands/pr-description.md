# Create a PR description

Check the diff against main, and produce a PR title and description.

## Title
Make this concise and fittable in a git commit's title. If the branch has a linear ticket defined, include that in [].

## Rest
The PR description should be produced in Markdown. Here's some high-level guidance on a good
description:

- Use ## for headings
- Put a high-level 1-3 sentence summary at the top, followed by a more detailed breakdown of the
  changes. Add this in a ## Summary section.
- Aim for brevity and avoid verbose descriptions of things that are manifestly clear from code.
- The description should focus on the "what", and less on the "how", with an emphasis on product and
  architectural changes
- If there are descriptive commit messages, reference them as a guide for what should be included in
  the description
- Add a testing section with checkboxes
